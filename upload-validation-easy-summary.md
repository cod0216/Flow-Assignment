# 파일 업로드 검증 쉬운 정리

## 핵심

- **이름(확장자)만 믿으면 위험**합니다.
- 파일 이름은 사용자가 쉽게 속일 수 있습니다.
- 그래서 **파일 내용까지 확인**해야 안전합니다.

## 1) 확장자 검사

- 보는 대상: 파일 이름
- 예: `cat.jpg`
- 한계: 이름은 바꿀 수 있음 (`virus.exe` -> `cat.jpg`)

## 2) MIME 검사

- 보는 대상: 파일 유형
- 예: 진짜 이미지인지, 실행파일인지
- 특징: 이름보다 신뢰도가 높음

## 3) 매직바이트 검사

- 보는 대상: 파일의 첫 바이트(시그니처)
- 예:
  - exe: `MZ`
  - pdf: `%PDF-`
- 특징: 실제 파일 내용 기반이라 가장 강력함

## 실무 권장 순서

1. 확장자 확인
2. MIME 확인
3. 매직바이트 확인
4. 하나라도 이상하면 업로드 차단

## 한 줄 요약

**이름만 보지 말고, 파일 내용까지 확인해야 안전합니다.**
